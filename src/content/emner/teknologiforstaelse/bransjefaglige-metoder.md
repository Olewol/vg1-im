# Bransjefaglige metoder og relevant utstyr

## 🎯 Hva skal du lære?

Du skal bli kjent med hvordan bransjen jobber — hvilke metoder, verktøy og arbeidsmåter som er vanlige i teknologi- og mediebransjen. Du lærer om prosjektarbeid, dokumentasjon, kvalitetssikring og hvordan du kan jobbe effektivt og profesjonelt.

---

## 📘 Fagstoff

### Hva er bransjefaglige metoder?

Bransjefaglige metoder er etablerte måter å jobbe på som er utviklet gjennom erfaring i bransjen. De sikrer kvalitet, effektivitet og forutsigbarhet i arbeidet.

**Hvorfor er metoder viktige?**
- Du slipper å finne opp hjulet på nytt
- Kvaliteten blir jevnere og mer forutsigbar
- Det blir lettere å samarbeide i team
- Kunder og kolleger vet hva de kan forvente

### Prosjektmetodikk

#### Tradisjonell (waterfall)
Lineær fremgangsmåte der hver fase fullføres før neste starter:
```
Kravspesifikasjon → Design → Utvikling → Testing → Leveranse
```
**Passer for:** Prosjekter med tydelige krav og lite endringer underveis (f.eks. bygg, store offentlige IT-systemer)

#### Smidig (agile)
Iterativ fremgangsmåte der man jobber i korte sykluser (sprinter):
```
Planlegg → Gjør → Test → Vurder → Gjenta
```
**Passer for:** Prosjekter der kravene endrer seg underveis (f.eks. app-utvikling, nettsider)

**Vanlige smidige rammeverk:**
- **Scrum:** Jobber i 1-4 ukers sprinter, med daglige standup-møter
- **Kanban:** Visuell styring av oppgaver på en tavle (trello, Notion)
- **Prosessmodellen:** Idé → Forarbeid → Produksjon → Etterarbeid → Vurdering

### Relevant utstyr i bransjen

**Opptaksutstyr (lyd og bilde):**
- Kameraer (DSLR, speilløse, systemkameraer)
- Mikrofoner (kondensator, dynamisk, shotgun, lavalier)
- Lysutstyr (softbox, panellys, ringlys)
- Stativer og gimbals

**Programvare:**
- Adobe Creative Cloud (Photoshop, Premiere, After Effects, Illustrator, InDesign)
- DaVinci Resolve (videoredigering)
- Figma (UI/UX-design og prototyping)
- VS Code / Git (utvikling og versjonskontroll)
- Slack / Teams (kommunikasjon)

**Produksjonsutstyr:**
- Mixere og lydgrensesnitt
- Stream-decks og produksjonskontrollere
- Green screen / chroma key-utstyr

### Dokumentasjon

Dokumentasjon er avgjørende i alle teknologiprosjekter. Uten dokumentasjon forsvinner kunnskap når folk slutter.

**Typer dokumentasjon:**
- **Teknisk dokumentasjon:** Hvordan ting er bygget (systemarkitektur, kildekode)
- **Brukerdokumentasjon:** Hvordan ting brukes (manualer, hjelpesider)
- **Prosjektdokumentasjon:** Planer, møtereferater, framdriftsrapporter
- **Prosessdokumentasjon:** Hvordan arbeidet ble utført (logg, rapport)

God dokumentasjon skal være:
- **Nøyaktig:** Riktig og oppdatert
- **Forståelig:** Tilpasset målgruppen
- **Tilgjengelig:** Lett å finne når man trenger den

### Kvalitetssikring

Kvalitetssikring handler om å sjekke at arbeidet holder mål:

| Metode | Hva det er | Eksempel |
|--------|-----------|----------|
| **Test** | Sjekke at noe virker | Teste en nettside i ulike nettlesere |
| **Gjennomgang** | Kolleger ser over arbeidet | Code review, design review |
| **Brukertest** | Målgruppen prøver løsningen | Elevene tester en app |
| **Sjekkliste** | Liste over krav som må være oppfylt | Kvalitetssjekk før publisering |

### Samarbeid og yrkesrolle

I teknologi- og mediebransjen jobber du sjelden alene. Samarbeid er nøkkelen:

**Verktøy for samarbeid:**
- **Versjonskontroll:** Git/GitHub — flere kan jobbe på samme kode uten å ødelegge for hverandre
- **Prosjektstyring:** Trello, Jira, Notion — følge oppgaver og framdrift
- **Fildeling:** Google Drive, OneDrive, Dropbox — dele og samarbeide om filer
- **Kommunikasjon:** Slack, Teams, Discord — holde kontakt med teamet

**Yrkesrollen din:**
- Vær løsningsorientert — fokuser på hva som kan gjøres, ikke på problemer
- Kommuniser tydelig — spør hvis noe er uklart
- Ta ansvar — eie oppgavene dine
- Vis initiativ — foreslå forbedringer

## 💡 Praktiske eksempler

**Eksempel 1: En arbeidsuke i et mediebyrå**
Mandag: Planleggingsmøte (scrum), fordel oppgaver på tavla
Tirsdag: Produksjon — film et intervju med lys og lyd
Onsdag: Redigering i Premiere Pro, kollega ser over klippet
Torsdag: Leveranse til kunde, få tilbakemelding
Fredag: Retting etter tilbakemelding, dokumenter hva som ble gjort

**Eksempel 2: Prosessmodellen for en skoleoppgave**
1. **Idé:** Hva skal vi lage? Hvem er målgruppen? Hva er budskapet?
2. **Forarbeid:** Research, skisser, manus, storyboard, tidsplan
3. **Produksjon:** Film/fotografer/programmer — gjennomfør planen
4. **Etterarbeid:** Rediger, fargekorriger, lydlegg, publiser
5. **Vurdering:** Hva gikk bra? Hva kunne vært bedre? Hva lærte du?
